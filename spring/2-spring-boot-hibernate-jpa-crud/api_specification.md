# Spring Boot Hibernate JPA CRUD — API Specification

## Description

`"spring/2-spring-boot-hibernate-jpa-crud/"` teaches **database persistence with Spring Data JPA and Hibernate** using the **JPA `EntityManager`** in a custom DAO layer. It does **not** expose HTTP endpoints — there are no `@RestController` classes in this module.

The **programmatic API** is:

- `"StudentDAO"` — CRUD contract for `"Student"` entities
- `"StudentDAOImpl"` — Hibernate/JPA implementation injected with `EntityManager`
- `"Application.java"` — `CommandLineRunner` demos that call the DAO on startup (console output, not REST)

**Prerequisites:**

- Java 21 (see `"pom.xml"`)
- MySQL running locally with database `student_tracker`
- DB user `springstudent` / `springstudent` (see `"sql-scripts/"`)
- Module 1 familiarity with Spring Boot wiring helps

For folder layout and learning path, see `"project_specification.md"`.

## Persistence API — `StudentDAO`

Implementation: `"src/main/java/com/lab/crud/demo/dao/StudentDAOImpl.java"`

Entity: `"src/main/java/com/lab/crud/demo/entity/Student.java"` → table `student` (`id`, `first_name`, `last_name`, `email`)

### Controller index (DAO methods)

| Method | Operation | Transaction | Teaches |
|--------|-----------|-------------|---------|
| `save(Student)` | INSERT | `@Transactional` | `entityManager.persist` |
| `findById(Integer)` | SELECT by PK | read-only | `entityManager.find` |
| `findAll()` | SELECT all | read-only | JPQL `FROM Student` |
| `findByLastName(String)` | SELECT with param | read-only | JPQL bind parameter |
| `update(Student)` | UPDATE | `@Transactional` | `entityManager.merge` |
| `delete(Integer)` | DELETE by PK | `@Transactional` | `find` + `remove` |
| `deleteAll()` | DELETE all rows | `@Transactional` | bulk JPQL `executeUpdate` |

---

### `save` — Create (INSERT)

- **Description**: Persists a new `Student` row; Hibernate assigns `id` after flush (identity column).
- **Signature**: `void save(Student student)`
- **Input**: `Student` with `firstName`, `lastName`, `email` (id unset or 0)
- **Output**: void; caller reads generated id via `student.getId()` after save
- **Dependency injection**: `StudentDAOImpl` receives `EntityManager` via constructor
- **Details**: `entityManager.persist(student)` inside `@Transactional`
- **Learn**: `persist` attaches a new entity; ID is generated by `GenerationType.IDENTITY` on `"Student.java"`

---

### `findById` — Read one (SELECT)

- **Description**: Loads a single student by primary key.
- **Signature**: `Student findById(Integer id)`
- **Input**: Primary key `id`
- **Output**: `Student` instance, or `null` if not found
- **Details**: `entityManager.find(Student.class, id)` — no custom JPQL
- **Learn**: `find` is the simplest PK lookup; always null-check in calling code

---

### `findAll` — Read all (SELECT)

- **Description**: Returns every student ordered by `lastName` ascending.
- **Signature**: `List<Student> findAll()`
- **Input**: None
- **Output**: `List<Student>` (possibly empty)
- **Details**: JPQL `"FROM Student order by lastName asc"` via `TypedQuery`
- **Learn**: Entity name in JPQL is the **class** name (`Student`), not the table name (`student`)

---

### `findByLastName` — Query with parameter (SELECT)

- **Description**: Finds students matching a last name exactly.
- **Signature**: `List<Student> findByLastName(String lastName)`
- **Input**: `lastName` string
- **Output**: `List<Student>` (possibly empty)
- **Details**: JPQL `WHERE lastName=:theData` with `query.setParameter("theData", lastName)`
- **Learn**: Named parameters avoid string concatenation and SQL injection in JPQL

---

### `update` — Update (UPDATE)

- **Description**: Merges changes from a detached or managed `Student` into the database.
- **Signature**: `void update(Student student)`
- **Input**: `Student` with id set and fields to change
- **Output**: void
- **Details**: `entityManager.merge(student)` inside `@Transactional`
- **Learn**: Typical flow — `findById`, mutate setters, then `update`; `merge` synchronizes state with the DB

---

### `delete` — Delete one (DELETE)

- **Description**: Removes one student by primary key.
- **Signature**: `void delete(Integer id)`
- **Input**: Primary key `id`
- **Output**: void; prints to console if id not found
- **Details**: `find` then `remove` if non-null; otherwise logs `"Entity with ID: … don't exist."`
- **Learn**: Must load the entity (or use bulk delete) before `remove` in standard JPA

---

### `deleteAll` — Bulk delete (DELETE)

- **Description**: Deletes all rows in the `student` table via bulk JPQL.
- **Signature**: `int deleteAll()`
- **Input**: None
- **Output**: `int` — number of rows deleted
- **Details**: `entityManager.createQuery("DELETE from Student").executeUpdate()`
- **Learn**: Bulk updates bypass the persistence context for each row — use carefully in production

## Startup demos — `CommandLineRunner`

Source: `"src/main/java/com/lab/crud/demo/Application.java"`

On boot, Spring runs the `commandLineRunner` bean. **Only one demo is active** unless you uncomment others in source.

| Demo method | Active by default? | What it does |
|-------------|-------------------|--------------|
| `createStudent` | No (commented) | Saves one `Student` ("Paul", "Wall", …) |
| `createMultipleStudents` | **Yes** | Saves three students (Doe, Smith, Johnson) |
| `findStudentById` | No | Loads student by id (example uses `7`) |
| `queryForAllStudents` | No | Prints all students |
| `queryFindStudentsByLastName` | No | Queries last name `"Doe"` |
| `updateStudent` | No | Changes first name for id `1` to `"Pepe"` |
| `deleteStudent` | No | Deletes id `1` |
| `deleteAllStudents` | No | Wipes `student` table |

**Learn**: Uncomment one demo at a time in `commandLineRunner` to exercise each DAO method and read SQL in the console (`logging.level.org.hibernate.SQL=debug`).

## Examples

### Prepare MySQL (first time)

Run scripts from `"sql-scripts/"` against your MySQL instance:

```bash
# Example — adjust mysql client path and credentials for your install
mysql -u root -p < "spring/2-spring-boot-hibernate-jpa-crud/sql-scripts/01-create-user.sql"
mysql -u root -p < "spring/2-spring-boot-hibernate-jpa-crud/sql-scripts/02-student-tracker.sql"
```

### Run the app (default demo: create 3 students)

From `"spring/2-spring-boot-hibernate-jpa-crud/"`:

```bash
./mvnw spring-boot:run
```

Windows:

```bash
.\mvnw.cmd spring-boot:run
```

**Expected console output (abbreviated):**

```text
Creating 3 new student objects...
Saved student: Student [id=1, firstName=John, lastName=Doe, email=john.doe@example.com] with id: 1
Saved student: Student [id=2, firstName=Jane, lastName=Smith, email=jane.smith@example.com] with id: 2
Saved student: Student [id=3, firstName=Bob, lastName=Johnson, email=bob.johnson@example.com] with id: 3
```

Hibernate SQL appears in logs because `logging.level.org.hibernate.SQL=debug`.

### Try another operation

In `"Application.java"`, comment `createMultipleStudents(studentDAO);` and uncomment e.g.:

```java

```

Restart the app to see all rows printed.

```bash
mysql -u springstudent -pspringstudent -e "USE student_tracker; SELECT * FROM student;"
```

## Concepts

| Concept | Where it appears |
|---------|------------------|
| JPA entity mapping | `"Student.java"` — `@Entity`, `@Table`, `@Column`, `@Id`, `@GeneratedValue` |
| No-arg constructor | Required by Hibernate for entity instantiation |
| `EntityManager` | Injected into `"StudentDAOImpl"` — core JPA API |
| `@Repository` | Marks DAO as a Spring persistence component |
| `@Transactional` | On write operations (`save`, `update`, `delete`, `deleteAll`) |
| JPQL | `findAll`, `findByLastName`, `deleteAll` queries |
| `persist` vs `merge` | Create vs update paths |
| `ddl-auto=create` | Hibernate recreates schema on startup (dev only) |
| `CommandLineRunner` | Runs DAO demos after context is ready |
| Constructor injection | `StudentDAOImpl(EntityManager)` |

### Entity ↔ table mapping

```java
@Entity
@Table(name = "student")
public class Student {
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    @Column(name = "id")
    private int id;

    @Column(name = "first_name")
    private String firstName;
    // ...
}
```

Field names use camelCase; column names use snake_case — Hibernate maps between them via `@Column(name = …)`.

## Configuration

File: `"src/main/resources/application.properties"`

| Property | Value | Effect |
|----------|-------|--------|
| `spring.datasource.url` | `jdbc:mysql://localhost:3306/student_tracker` | MySQL database |
| `spring.datasource.username` | `springstudent` | DB user |
| `spring.datasource.password` | `springstudent` | DB password |
| `spring.jpa.hibernate.ddl-auto` | `create` | Drop/recreate tables on each startup — **not for production** |
| `logging.level.org.hibernate.SQL` | `debug` | Print SQL statements |
| `logging.level.org.hibernate.orm.jdbc.bind` | `trace` | Print bind parameter values |

Dependencies (`"pom.xml"`): `spring-boot-starter-data-jpa`, `mysql-connector-j` (runtime). **No** `spring-boot-starter-web` — confirms no HTTP layer.

## Notes / Caveats

- **No REST API:** This module has no HTTP endpoints. For REST documentation patterns, see `"spring/1-spring-boot-setup/api_specification.md"` or `"spring/3-spring-rest/"`.
- **`ddl-auto=create`:** Schema is recreated on every startup — data from previous runs is lost unless you change this setting.
- **Demo id assumptions:** `updateStudent` / `deleteStudent` examples use id `1` — only works if that row exists after your create demo.
- **MySQL required:** App fails to start if MySQL is not reachable at `localhost:3306`.
- **Next step:** A natural follow-up is adding a `@RestController` that delegates to `StudentDAO` — that would be a separate learning exercise or module.
