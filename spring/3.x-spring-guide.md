# Spring Course Modules Summary

## 3.1 — spring-sample-rest

### Features
- `/test/hello`
- `/api/students`
- Global error handling with `@ControllerAdvice`

### Teaching Comments
- In-memory list index pitfall
- `@PostConstruct`
- `@ControllerAdvice`

### Important Notes
- `{studentId}` is a **list index (0–2)**, **not a database ID**.

---

## 3.2 — spring-jpa-api-rest

### Features
- Full CRUD at `/api/employees`
- Manual `EmployeeDAO` implementation

### Teaching Comments
- REST architecture
- Manual DAO pattern
- `EntityManager.merge()`
- PATCH implementation using `JsonMapper`

### Important Notes
- Class name `EmployeeControler` (typo) is intentionally called out in the specs.

---

## 3.3 — spring-data-jpa

### Features
- Same REST API as **3.2**
- Backend powered by `JpaRepository`

### Teaching Comments
- `JpaRepository` vs manual DAO approach from **3.2**

### Important Notes
- Class name `EmployeeControler` (typo) is intentionally called out in the specs.

---

## 3.4 — spring-data-rest

### Features
- No controller required
- Auto-generated REST API at `/api/members`
- HAL-based responses

### Teaching Comments
- `@RepositoryRestResource(path = "members")`
- No controller layer

### Important Notes
- URLs are **`/api/members`**, not `/api/employees`.

---

## 3.5 — spring-data-jpa-swagger

### Features
- Same REST API as **3.3**
- Swagger/OpenAPI integration
- Swagger UI available at:
  - `http://localhost:8080/api/swagger-ui.html`

### Teaching Comments
- SpringDoc integration
- Same REST implementation as **3.3**

### Important Notes
- Open `http://localhost:8080/api/swagger-ui.html` while the application is running.
- Class name `EmployeeControler` (typo) is intentionally called out in the specs.

---
